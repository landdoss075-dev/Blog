## [2026-06-03] — Реализация автопостера

### Решено (Decisions)
- **Генератор текста**: окончательно выбран **Groq (Llama 3.3 70b)**, а не Claude.
  В документации было расхождение (стек указывал Groq, а пример workflow — `ANTHROPIC_API_KEY`).
  Groq бесплатный и под него расписана вся инструкция → пайплайн использует `GROQ_API_KEY`.
- **Структура кода**: модульная (`src/` + тонкий оркестратор), чтобы легче добавлять VK/TenChat.
- **Отказоустойчивость**: площадки без токена мягко пропускаются; одна упавшая площадка
  не роняет вторую. Запуск считается провальным, только если не опубликовала ни одна.
- **Dry-run**: флаг `--dry-run` / `DRY_RUN=1` — генерирует статью в `out/` без публикации.

### Создано
- `package.json`, `package-lock.json`, `.gitignore`, `.env.example`, `README.md`
- `src/config.js`, `src/log.js`, `src/news.js`, `src/groq.js`, `src/unsplash.js`,
  `src/telegram.js`, `src/zen.js`
- `scripts/generate_and_post.js` — оркестратор пайплайна
- `.github/workflows/daily_post.yml` — cron 10:00 МСК + ручной запуск (с опцией dry-run)

### Проверено
- Зависимости ставятся (`npm install`), модули грузятся без ошибок.
- Парсинг Google News RSS работает — тема дня подтягивается.
- Проверка обязательного `GROQ_API_KEY` срабатывает с понятной ошибкой.

## [2026-06-03] — Умный отбор темы (свежесть + тренд)

### Решено (Decisions)
- **Формат контента**: «польза на поводе», а не голый рерайт новости. Горячая тема —
  зацепка в заголовке/первом абзаце, дальше практическая ценность (советы, кейсы, промпты).
  Причина: чистый рерайт занижается лентой Дзена и проигрывает СМИ по скорости; дочитывания
  (порог 10к минут) дают полезные материалы, а не пересказы.
- **Отбор темы**: свежесть (≤48ч) + частотный «трендоскоп» — самая часто упоминаемая тема
  среди свежих заголовков как прокси популярности.

### Изменено
- `src/news.js` переписан: тянет поисковые запросы + топ техно-ленту Google News,
  дедуплицирует, фильтрует по свежести, считает частоту тем. Грубый стемминг схлопывает
  словоформы; слова из собственных запросов исключаются как шум (всплывает реальный повод).
- `src/groq.js`: промпт перестроен под формат «польза на поводе», получает тренды дня.

### Проверено
- Отбор темы работает на живых данных: из общего шума («искусственный интеллект»)
  корректно всплывает конкретный горячий повод дня с читаемыми тренд-словами.

## [2026-06-04] — Провайдеры, проверка API через веб, сайт+RSS для Дзена

### Решено (Decisions)
- **Генератор**: выбран **OpenRouter `openai/gpt-5-mini`** (один ключ на все модели).
  Сравнение на одной теме показало: Llama 3.3 70b даёт шаблонный текст с водой,
  gpt-5-mini — конкретные шаги, готовые промпты, кейсы, цифры. Цена ~0.4¢/статья (~$1.5/год).
- **❌ Zen Publisher API не существует** (проверено вебом). Публикация в Дзен — только
  через **импорт RSS-ленты с сайта на GitHub Pages**. См. память `zen-no-publish-api`.
- **Старт обеих площадок сразу**: Telegram (Bot API) + сайт/Дзен (RSS).

### Добавлено
- Провайдер-свитч: `src/prompt.js` (общий промпт/парсинг), `src/groq.js`, `src/openai.js`,
  `src/openrouter.js`, `src/llm.js` (роутер по `PROVIDER`). `gpt-5/o-серия`: учтены
  `reasoning_effort`/`max_completion_tokens` (без этого reasoning съедал весь лимит → пустой ответ).
- `src/sanitize.js` — чистка HTML от модели (чинит битые теги вроде `<pМногие`, белый список).
- `src/site.js` — генерация статического сайта (GitHub Pages) + Дзен-совместимой RSS-ленты
  (namespaces yandex/media, content:encoded, yandex:full-text, enclosure, pubDate RFC-822).
- `scripts/compare_providers.js` + `npm run compare` — сравнение моделей на одной теме.
- Workflow: права `contents: write` + шаг коммита сгенерированного `docs/` обратно в репо.

### Удалено
- `src/zen.js` (фиктивный Zen API).

### Проверено (вживую)
- OpenRouter gpt-5-mini end-to-end: тема → статья → сайт+RSS. Статья качественная.
- RSS валиден и соответствует требованиям Дзена; санитайзер отработал (HTML чистый).
- Стоимость/имя модели OpenRouter сверены вебом ($0.25/$2 за 1M, slug верен).

## [2026-06-04] — Темп ×3, защита от дублей, оптимизация под Дзен, бренд, Дзен-канал

### Темп и качество тем
- **Cron 3×/день** (10:00 / 15:00 / 20:00 МСК).
- **Защита от повторов** (`news.js`): если день «захвачен» мега-событием (≥3 заголовков)
  и о нём уже писали (из `docs/posts.json`) — берём повод из другого кластера.
  Тема промпта берётся из выбранного заголовка, а не глобальных трендов.

### Оптимизация под монетизацию Дзена 2026
> Актуальные правила (проверено вебом): монетизация = **30 часов вовлечённого чтения от
> ПОДПИСЧИКОВ за 30 дней** (~5 коп/мин). Метрика «10 000 минут дочитываний» — устарела.
- **Фильтр чувствительных тем** (`news.js`): отсев политики/войны/трагедий/насилия — Дзен за
  это снижает/отключает монетизацию. Держимся чистых ИИ-инструментов. (отсеивает ~20+ тем/день)
- **CTA-подписка + вопрос к читателю** в конце статьи (`prompt.js`) — рост подписчиков и комментов.
- **2-3 картинки внутри текста** (`unsplash.js fetchImages` + `site.js insertInlineImages`) —
  больше минут чтения.
- Всё проверено вживую на тестовом прогоне (фильтр + CTA + картинки + вопрос — работают).
