## [2026-06-03] — Умный отбор темы (свежесть + тренд)

### Решено (Decisions)
- **Формат контента**: «польза на поводе», а не голый рерайт новости. Горячая тема —
  зацепка в заголовке/первом абзаце, дальше практическая ценность (советы, кейсы, промпты).
  Причина: чистый рерайт занижается лентой Дзена и проигрывает СМИ по скорости; дочитывания
  (порог 10к минут) дают полезные материалы, а не пересказы.
- **Отбор темы**: свежесть (≤48ч) + частотный «трендоскоп» — самая часто упоминаемая тема
  среди свежих заголовков как прокси популярности.

### Изменено
- `src/news.js` переписан: тянет поисковые запросы + топ техно-ленту Google News,
  дедуплицирует, фильтрует по свежести, считает частоту тем. Грубый стемминг схлопывает
  словоформы; слова из собственных запросов исключаются как шум (всплывает реальный повод).
- `src/groq.js`: промпт перестроен под формат «польза на поводе», получает тренды дня.

### Проверено
- Отбор темы работает на живых данных: из общего шума («искусственный интеллект»)
  корректно всплывает конкретный горячий повод дня с читаемыми тренд-словами.

## [2026-06-30] — Уход от шаблона под Дзен 2026 + ссылка на Telegram

### Повод
Разбор «зашедшей» статьи (2766 показов в кабинете Дзена) показал: это не успех контента,
а **разовый тест канала** Дзеном — и контент его провалил (CTR 1.4% при норме 2-4%,
дочитывание ~15% при норме 40%+, 0 реакций). На фоне остальных статей по 2-19 показов.

### Сверено вебом (механика Дзена 2026)
- Новую статью Дзен тестирует на ~300-1000 чел; расширяет охват «снежным комом» только при
  хороших CTR+дочитываниях (цикл 24-72ч). **Кол-во публикаций само по себе охват не даёт.**
- Платит за минуты вовлечённого чтения подписчиков (~0.05 ₽/мин), не за показы/число статей.
- **Главный риск:** в 2026 алгоритм целенаправленно ДЕТЕКТИТ и режет шаблонный машинный текст.
  ИИ-текст без ручной доработки формально не разрешён.

### Корневая проблема
Все статьи — клоны по структуре («Зацепка:→проблема→3 шага→CTA»), заголовки — штамп
«[новость] — как сделать…» через тире, объём ~480-500 слов. Ровно то, что Дзен 2026 давит.

### Изменено (D:\Blog, коммит 0808448)
- `prompt.js`: ротация **6 форматов подачи** + **5 стилей заголовка** (рандом на запуск),
  запрет маркеров «Зацепка»/«Конкретные шаги» и тире-штампа, объём **700-1100 слов**,
  **3 варианта заголовка** (`titles[]`; берём первый, остальные в `titleVariants`),
  проброс недавних заголовков (не повторять формулировки), конкретный `image_query`.
  Новая функция `qualityIssue()` (бар: ≥2 заголовка, тело ≥400 слов).
- `llm.js`: **один ретрай** генерации, если статья не дотянула — отсекает осечки старого
  формата (~180 слов / 1 заголовок).
- `news.js`: `fetchTopic` возвращает `recentTitles`.

### Кросс-промо: ссылка на Telegram «Нейробудни»
- Гиперссылка (`<a href>`) в финальном CTA-блоке `prompt.js` — **одна, в конце, после пользы**
  (Дзен за такое не штрафует; режут за навязчивые призывы + слабый контент). Добавляется
  ПОСЛЕ санитайзера, иначе `<a>` вычистился бы из тела модели.
- URL через `TELEGRAM_CHANNEL_URL` (config + GitHub Actions var + workflow env;
  дефолт `https://t.me/ai_news_blog`). При смене @username канала — меняется одна переменная.
- В Telegram-пост (`article.telegram`) ссылку на свой же канал НЕ добавляем.
- Проверена вся цепочка до Дзена: `insertInlineImages` не трогает → atmoapp шлёт content как
  есть → RSS-route кладёт в `content:encoded` через CDATA без пересанитайза. Доходит целым.

### Решения (осознанно НЕ сделано)
- **НЕ переходить на 3/день в Дзен** — снизит охват (детект шаблона + каннибализация).
  Telegram=3/день ок (хронолента), Дзен=1/день через RSS atmoapp — правильно как есть.
- **«Лучшая из 3» в Дзен НЕ делаем** — сейчас едет первая за сутки (утренняя 07:00 UTC),
  дневная/вечерняя в БД atmoapp не попадают (`ingest` при `todayCount>=1` → skipped).
  Автовыбор лучшей невозможен без сигнала реакции (дочитывания приходят через часы, доступа
  нет) — мерить можем только форму (`bodyWords`), что метрику почти не двигает. Ретрай уже
  гарантирует, что утренняя нормального качества. Настоящая «лучшая из 3» — только когда
  появится сигнал реакции (напр. парсить просмотры Telegram-постов через сутки).

### Проверено
- Живые dry-run: статьи стали структурно разными, заголовки — кликабельные без тире-штампа,
  объём 620-640 слов, гиперссылка на «Нейробудни» в теле, в Telegram-поле ссылки нет.
- Задеплоено: rebase поверх прод-коммитов, push в `main`; переменная заведена в Actions.

## [2026-07-02] — Telegram: оформленные посты через Rich Messages (Bot API 10.1)

### Что сделано
Посты в Telegram-канал теперь выходят оформленными «как на Дзене»: заголовок, подзаголовки,
абзацы, списки, жирный, кликабельная ссылка на «Нейробудни». Раньше уходил обрезанный
plain-caption (поле `telegram`, 600-900 симв.) — теперь полное тело статьи с вёрсткой.

### Ключевое открытие (сэкономило кучу кода)
`InputRichMessage` — это НЕ дерево блоков (`{blocks:[...]}`), как я сперва понял из списка
классов RichBlock*. Реальная схема (проверено по доке + живым вызовом): `InputRichMessage`
принимает **ГОТОВУЮ HTML-СТРОКУ** в поле `html` (либо `markdown`): «Exactly one of the
fields html or markdown must be used». Классы RichBlock*/RichText* — это внутреннее
представление ПОЛУЧАЕМЫХ сообщений, для отправки не нужны. Наш `article.html` подошёл почти
как есть → конвертер в блоки не понадобился (был написан и удалён).
Отладка: `sendRichMessage` с `{blocks}` всегда давал `rich message must be non-empty`
(Telegram искал html/markdown). С `rich_message:{html:"<h2>…</h2><p>…</p>"}` — сразу OK.

### Реализация (`src/telegram.js`)
- Обложка — ГОЛАЯ картинка (`sendPhoto` без caption), т.к. заголовок+тело идут в Rich
  следом; анонс под фото убрали (дублировал статью — проверено на рендере).
- Тело — `sendRichMessage` с `rich_message.html` = `<h1>заголовок</h1>` + `article.html`
  (со всей вёрсткой и CTA-ссылкой) + теги. Лимит Rich — 32768 симв.
- Все наши теги приняты rich_message.html: h1/h2/h3/p/ul/ol/li/b/strong/i/em/br/a (проверено).
- **Фолбэк:** если `sendRichMessage` упадёт (фича сырая/клиент не тянет) — досылаем тело
  обычным текстом (h2→<b>, li→«•», разбивка по 4096). Прод не сломается.

### Проверено
- Живые отправки в личку (@hackyoou, chat_id 630393753): Rich `rich:true`, статья 574 слова
  отрендерилась с заголовками/списками/ссылкой. Пользователь подтвердил рендер на телефоне — ок.
- Обложка голая + одно Rich-сообщение = цельный пост без дублей.

### Действует
- Со следующего cron-поста в канал @ai_news_blog. Дубль записи «на будущее» выше — закрыт.

## [2026-07-16] — Фиксы: битый JSON от модели + один пост в Telegram

### Фикс 1: генератор падал на битом JSON (утренний пост не выходил)
Health-check прислал алерт «Ежедневная статья упала» (07:00 UTC). Причина: Sonnet 5
иногда отдаёт JSON с сырыми переводами строк/табами внутри строковых значений
(html/telegram) → `parseArticle` бросал `SyntaxError: Bad control character`. Ретрай
ловил только качество, но НЕ исключение парсинга → битый ответ ронял весь процесс.
- `src/prompt.js`: `extractJson` при ошибке чинит сырые \n/\r/\t внутри строковых
  значений и парсит повторно (посимвольный проход с отслеживанием «внутри строки»).
- `src/llm.js`: `parseArticle` в try/catch, до 3 попыток. Битый ответ = перегенерировать.

### Фикс 2: в Telegram выходило 2 поста → теперь 1 (картинка внутри Rich)
Раньше: обложка (sendPhoto) отдельным сообщением + статья (sendRichMessage) отдельным
→ разорванно. Теперь один пост: обложка встроена в Rich.
- КАК: `InputRichMessage` имеет поле `media` (Array of InputRichMessageMedia). В html
  ставим `<img src="tg://photo?id=cover">`, в media — `[{id:'cover', media:{type:'photo',
  media:<URL>}}]`. `InputMediaPhoto.media` принимает HTTP-URL напрямую — Telegram сам
  скачивает картинку, file_id НЕ нужен (это и был блокер «RichBlockPhoto требует file_id»).
- `src/telegram.js`: один `sendRichMessage` с media[] вместо sendPhoto+sendRichMessage.
  Фолбэк (если Rich упадёт) сам шлёт фото+текст обычным постом.
- Инлайн-картинки статьи (из atmoapp) в TG-посте не показываются (нужна привязка через
  media[] по id) — для авто-постов это не важно (обложка идёт из image, тело без инлайн-img).

### Проверено
- Оба фикса: dry-run зелёный, юнит-тест repair на реальной поломке. Telegram — живые

## [2026-07-16] — Проект «5 автономных ниш»: ресёрч, план, архитектурное решение

### Идея (пользователь)
Запустить 5 новых Дзен-каналов в читаемых нишах. Каждая ниша — автономная экосистема:
свой SEO-сайт (с него RSS в Дзен) + свой Telegram-канал + свой Дзен-канал.
Сайт — самостоятельный SEO-актив, а не «труба» для ленты.
План-артефакт: https://claude.ai/code/artifact/e42cba67-56ac-4045-a911-3f44e33ea5c4

### Ресёрч аудитории и ниш (сверено вебом, 2 итерации)
- **Аудитория Дзена:** ядро 35-55 лет, 34% — старше 55 (крупнейшая группа), женщин 53%,
  ~26 мин чтения/день, 70% мобайл. Пишем для 45-60+, не для молодых айтишников.
- **Официальный рейтинг интересов Дзена:** наука/природа 58%, история 55%, (политика 54% —
  нельзя из-за монетизации), питомцы 53%, психология 52%. Наши ниши = 4 из 5 верхних строк.
- **Триггеры дочитываний** (кейсы практиков 2026): ностальгия, бытовые сравнения, личный
  опыт, злободневность с авторской интерпретацией. Среднее дочитывание 1-2.5 мин.
- **Сквозной формат-чемпион — «история из жизни»:** нарратив с героем и развязкой
  дочитывают в разы лучше советов списком. Вшивается в промпт-персону каждой ниши.

### Выбранные 5 ниш (с углами подачи под реальную аудиторию)
1. **Бытовые финансы 45+** — пенсии/льготы/ЖКХ/вычеты/наследство, НЕ инвестиции. Топ-CPM.
2. **Психология и отношения** — семейные/жизненные истории с разбором, не «5 советов».
3. **Дача/сад/огород** — практика + истории дачников. Безопаснейшая ниша. Кандидат на пилот.
4. **Питомцы** — истории спасения/судьбы + уход. Без вет-диагнозов.
5. **История с ностальгией** — быт/вещи/цены/судьбы («как жили»), не сухие даты.
Отсечены: кулинария (AI-рецепты без теста = шаблон-детект; кандидат №6 через «истории»),
авто (конкуренция+шаблонность), здоровье (Дзен режет мед-советы), рыбалка (нет новостей).

### SEO-требования к сайтам (сверено: SEO-2026)
- AI-контент НЕ под фильтром сам по себе (Google не гоняет AI-детекторы) — важны польза,
  E-E-A-T, Information Gain. Массовая генерация без доработки не работает.
- На каждой странице: автор с фото/био (без подписи автора нет «экспертного кредита»),
  Schema.org Article, OG, уникальные title/description, внутренние ссылки.
- sitemap.xml + IndexNow-пинг (переиспользуем опыт atmoapp) → индексация за часы.

### Требования Дзена к RSS-импорту (сверено)
Домен подтверждается HTML-файлом/метатегом ПО КОРНЮ; домен RSS = подтверждённый домен;
10 подписчиков + 10 материалов до автоимпорта; проверок ленты ≤3/год; IDN-домены нельзя.
До 10 подписчиков — Дзен наполняется вручную (путь основного канала).

### ✅ Архитектурное решение: в ТЕКУЩЕМ проекте, не отдельным
Обсуждён вопрос «отдельный проект или в D:\Blog». Решение: **движок один — в D:\Blog**.
- Ниша = запись в `niches.js` (newsQueries, persona, site-таргет, telegram, sensitive-фильтр).
  «Нейробудни» станет нишей №1 (+ особый таргет atmoapp), новые — №2-6.
- Отдельный проект = два движка = дублирование всех выстраданных фиксов (анти-шаблон,

- **Граница: код ≠ контент.** 5 сайтов = 5 отдельных КОНТЕНТ-репозиториев (GitHub Pages

- Риск «сломать рабочий канал» управляем: рефакторинг поэтапно с dry-run, поведение
