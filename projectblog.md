# AI Tools Blog — Project Plan

## Идея проекта

Автоматический медиа-проект на тему **ИИ-инструментов** для русскоязычной аудитории.
Контент генерируется на автомате, публикуется одновременно на нескольких площадках.
Цель — монетизация через рекламу (Яндекс Дзен + Telegram).

---

## Архитектура (без сервера)

```
GitHub Actions (бесплатно, cron каждый день 10:00 МСК)
    ↓
Node.js скрипт (генерация + публикация)
    ↓
OpenRouter (gpt-5-mini) → генерирует статью про ИИ-инструменты
    ↓
├── Telegram канал — Bot API (пост с картинкой)
└── Сайт на GitHub Pages + RSS → импорт в Яндекс Дзен
```

**Сервер не нужен.** Всё работает через GitHub Actions бесплатно.

## Расширение: «5 автономных ниш» (план утверждён 16.07.2026, старт не начат)

Помимо основного канала («Нейробудни», ИИ-тематика) — 5 новых ниш. Каждая ниша =
автономная экосистема: **свой SEO-сайт → с него RSS в свой Дзен-канал + свой Telegram**.
Сайт — самостоятельный SEO-актив (E-E-A-T: автор с био, Schema.org, sitemap+IndexNow),
а не просто источник ленты. План-артефакт:
https://claude.ai/code/artifact/e42cba67-56ac-4045-a911-3f44e33ea5c4

**Ниши (углы подачи — под реальную аудиторию Дзена: ядро 35-55, треть 55+, женщин 53%):**
1. Бытовые финансы 45+ (пенсии/льготы/вычеты, НЕ инвестиции) — топ-CPM
2. Психология и отношения — через семейные/жизненные истории
3. Дача/сад/огород — практика + истории дачников (кандидат на пилот)
4. Питомцы — истории спасения + уход, без вет-диагнозов
5. История с ностальгией — быт/вещи/цены/судьбы, не сухие даты

**Сквозной формат всех ниш** — «история из жизни» (нарратив дочитывают в разы лучше
советов); триггеры: ностальгия, бытовые сравнения, личный опыт.

**Архитектура (решено):** движок ОДИН — в этом репозитории. Ниша = запись в `niches.js`
(newsQueries, persona, site-таргет, telegram, sensitive-фильтр); «Нейробудни» станет
нишей №1. Код ≠ контент: 5 сайтов — отдельные контент-репозитории (GitHub Pages в корне
домена — требование верификации Дзена), генератор пушит туда статику. Отдельный проект
отвергнут — два движка = дублирование всех фиксов.

**Порядок:** пилот на 1 нише → тираж конфигом. Темп 1 статья/день/ниша (~$5-6/мес Sonnet 5).
Подробности, требования Дзена/SEO и открытые вопросы — в changelogblog.md [2026-07-16].

- Срок до первых выплат: ~2-4 месяца при активном росте

    workflows/

### Логика скрипта:
1. Парсинг Google News RSS по ИИ-тематике → актуальные темы

5. Публикация в Яндекс Дзен через Zen Publisher API

## Требования к контенту (важно для Дзен)

- Живой язык, не шаблонный
- Конкретные примеры и кейсы
- Объём: 700-1100 слов (обновлено 30.06.2026 — больше минут чтения)
- Уникальность: каждая статья на новую подтему
- НЕ упоминать что текст написан ИИ

### ⚠️ Анти-шаблон под Дзен 2026 (обновлено 30.06.2026)

> Проверено вебом + по статистике канала: Дзен 2026 **целенаправленно детектит и режет
> охваты шаблонному машинно-сгенерированному тексту**. Раздаёт «снежным комом» по CTR
> (норма 2-4%) и дочитыванию (норма 40%+); платит за минуты вовлечённого чтения подписчиков
> (~0.05 ₽/мин), НЕ за число публикаций. Поэтому единая структура для всех статей —
> главный риск пессимизации.

- **НЕТ единой структуры.** Генератор ротирует 6 форматов подачи (личная история / разбор
  кейса / разбор ошибок / сравнение инструментов / связный гайд / разбор мифа) — рандом на запуск.
- **Запрещены маркеры-штампы:** «Зацепка», «Суть проблемы», «Конкретные шаги», «Что реально
  работает», механическая нумерация «Шаг 1/2/3».
- **Заголовки:** 3 варианта, ротация 5 стилей; ЗАПРЕЩЁН штамп «[новость] — как сделать…»
  через тире (им забита лента → низкий CTR).
- **Контроль качества:** ретрай генерации, если статья не дотянула (<2 заголовков / тело <400 слов).
- Реализация: `src/prompt.js` (FORMATS, TITLE_STYLES, `qualityIssue`), `src/llm.js` (ретрай),
  `src/news.js` (`recentTitles`). Подробности — память `dzen-content-strategy.md`.

### Ссылки в статьях Дзена (обновлено 30.06.2026)

- 1-2 внешние ссылки на полезный контент — норма, охват не режут. Режут за НАВЯЗЧИВЫЕ призывы
  «подпишись на ТГ» + слабый контент (спам/увод трафика).
- **Можно:** одна ссылка в КОНЦЕ статьи, после пользы. Реализовано — гиперссылка на Telegram
  «Нейробудни» в финальном CTA (`prompt.js`), URL через `TELEGRAM_CHANNEL_URL`.

### Темп публикаций по площадкам

- **Telegram** — 3/день (хронологическая лента, больше касаний — ок).
- **Дзен** — 1/день через RSS atmoapp (НЕ 3/день: усилит детект шаблона + каннибализирует охват).
- В Дзен едет первая статья за сутки (утренняя). «Лучшая из 3» не делаем — нет программного
  сигнала реакции для автовыбора (см. `dzen-content-strategy.md`).

## Инструкция: Groq API (бесплатная генерация текста)

### Что такое Groq
Groq — бесплатный AI API на базе модели Llama 3.3 70b от Meta. Работает быстрее Claude и GPT, бесплатный лимит: **14 400 запросов в день** — нам хватит на годы вперёд (1 запрос/день).

### Как получить ключ
1. Зайти на **console.groq.com**
2. Зарегистрироваться через Google аккаунт
3. Перейти в **API Keys → Create API Key**
4. Скопировать ключ — это и есть `GROQ_API_KEY`

### Как используется в скрипте
```js
const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.GROQ_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    model: 'llama-3.3-70b-versatile',
    messages: [{ role: 'user', content: prompt }],
    max_tokens: 2048,
  }),
});
const data = await response.json();
const text = data.choices[0].message.content;
```

### Бесплатные лимиты
| Лимит | Значение |
|---|---|
| Запросов в минуту | 30 |
| Запросов в день | 14 400 |
| Токенов в минуту | 6 000 |
| Цена | **$0** |

## Инструкция: публикация в Яндекс Дзен через RSS

> ⚠️ **ВАЖНО (проверено 2026):** у Яндекс Дзена **НЕТ открытого API для автопубликации**.
> Эндпоинт `api.zen.yandex.ru/v1/publisher/articles` из ранней версии плана не существует.
> Единственный штатный способ автопубликации — **импорт RSS-ленты с собственного сайта.**

### Архитектура решения
Скрипт-автопостер генерирует статьи как страницы статического сайта на **GitHub Pages**
(бесплатно, без сервера) и обновляет **RSS-ленту** (`docs/rss.xml`). Дзен подключается
к этой ленте и импортирует новые статьи автоматически. Бонус: сайт = третья площадка.

### Шаг 1 — Создать канал на Яндекс Дзен
1. Зайти на **dzen.ru** → «Создать канал» → тип **Блог**.
2. Заполнить название, описание, тематику «Технологии».

### Шаг 2 — Включить GitHub Pages
1. В репозитории: **Settings → Pages**.
2. Source: **Deploy from a branch**, branch `main`, папка **/docs**.
3. URL сайта будет вида `https://USERNAME.github.io/REPO` — это `SITE_URL`.

### Шаг 3 — Подключение RSS к Дзену (⚠️ реальные требования, проверено 06.2026)

**Блокеры, которые надо учитывать:**
1. **Нужно ≥10 ПОДПИСЧИКОВ канала** (не статей!) — только тогда Дзен разрешает импорт с сайта.
2. **Подтверждение владения идёт по КОРНЮ домена** (`USERNAME.github.io`, HTML-файл или мета-тег),
   а наш сайт сейчас на подпути `/Blog` → проверка по корню не пройдёт.
   **Домен RSS-ленты должен совпадать с подтверждённым доменом.**
3. Проверок RSS-ленты — **не более 3 в год** (подключать наверняка).

**Двухфазный план:**
- **Фаза 1 (сейчас → 10 подписчиков):** Telegram на автомате; Дзен наполняем **вручную** —
  копируем готовые статьи с сайта в редактор Дзена (ручная публикация без порогов).
  Кросс-промо Telegram↔Дзен для роста подписчиков.
- **Фаза 2 (≥10 подписчиков):** перенести сайт в КОРЕНЬ — репозиторий → `USERNAME.github.io`,
  сайт на `https://USERNAME.github.io/` (без `/Blog`). Добавить мета-тег `yandex-verification`
  в генератор (`site.js`). В Дзене: «Свой сайт» → подтвердить домен → «Настроить трансляцию
  материалов сайта» → указать `https://USERNAME.github.io/rss.xml`. Дальше Дзен сам импортирует.

**Монетизация Дзена 2026:** платят за **30 часов вовлечённого чтения от подписчиков за 30 дней**
(~5 коп/мин). Поэтому ключ — рост подписчиков и время чтения (отсюда CTA-подписка, вопросы,
картинки в тексте, безопасные темы).

### Требования Дзена к RSS (учтены в `src/site.js`)
- namespaces `yandex` / `media`, `content:encoded` (полный HTML) или `yandex:full-text`;
- `title` (≤200), `link` (≤243 ASCII), `pubDate` (RFC-822), картинка через `enclosure`;
- актуальность материалов — последние 8 дней, файл ленты ≤10 МБ.

## Инструкция: GitHub Actions (cron без сервера)

### Шаг 1 — Создать репозиторий
1. Зайти на **github.com** → **New repository**
2. Название: `ai-blog-autoposter` (или любое)
3. Тип: **Private** (чтобы скрипты и конфиги не были публичны)
4. Нажать **Create repository**

### Шаг 2 — Добавить API-ключи в Secrets
Это безопасное хранилище — ключи не видны никому, даже тебе после сохранения.

1. В репозитории: **Settings → Secrets and variables → Actions**
2. Нажать **New repository secret**
3. Добавить по одному:

**Secrets** (Settings → Secrets and variables → Actions → *Secrets*):

| Name | Value |
|---|---|
| `OPENROUTER_API_KEY` | ключ OpenRouter (генерация текста) |
| `UNSPLASH_ACCESS_KEY` | ключ Unsplash (опц.) |
| `TELEGRAM_BOT_TOKEN` | токен Telegram-бота |
| `TELEGRAM_CHANNEL_ID` | `@название_канала` |

**Variables** (там же, вкладка *Variables* — не секретные):

| Name | Value |
|---|---|
| `PROVIDER` | `openrouter` |
| `OPENROUTER_MODEL` | `openai/gpt-5-mini` |
| `SITE_URL` | `https://USERNAME.github.io/REPO` |
| `SITE_TITLE` / `SITE_DESCRIPTION` | название и описание сайта |

### Шаг 3 — Создать файл расписания
Создать файл `.github/workflows/daily_post.yml` в репозитории:

```yaml
name: Daily AI Post

on:
  schedule:
    - cron: '0 7 * * *'   # каждый день в 07:00 UTC = 10:00 МСК
  workflow_dispatch:        # ручной запуск для теста

jobs:
  post:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Run post generator
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          UNSPLASH_ACCESS_KEY: ${{ secrets.UNSPLASH_ACCESS_KEY }}
          TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
          TELEGRAM_CHANNEL_ID: ${{ secrets.TELEGRAM_CHANNEL_ID }}
          ZEN_PUBLISHER_TOKEN: ${{ secrets.ZEN_PUBLISHER_TOKEN }}
        run: node scripts/generate_and_post.js
```

### Шаг 4 — Проверить работу
1. После пуша файла зайти в **Actions** вкладку репозитория
2. Нажать **Run workflow** (ручной запуск для теста)
3. Смотришь логи — всё должно пройти без ошибок
4. Дальше GitHub сам запускает каждый день в 10:00 МСК

> **Бесплатный лимит GitHub Actions**: 2000 минут/месяц.
> Один запуск скрипта ≈ 1-2 минуты. На месяц уйдёт ~60 минут — в 30 раз меньше лимита.
